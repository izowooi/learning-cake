# 반응속도 테스트 웹 앱 PRD

## 프로젝트 개요
- **프로젝트명**: 반응속도 테스트 (Reaction Time Test)
- **기술 스택**: Next.js (App Router)
- **데이터베이스**: 없음 (클라이언트 사이드 상태 관리만 사용)
- **배포**: Cloudflare Pages
- **반응형**: 모바일 필수 지원

---

## 페이지 구성

### 1️⃣ 첫 번째 페이지 - 시작 화면

#### 구성 요소
- **제목**: "반응속도 테스트"
- **설명**: "당신의 반응속도를 테스트해보세요."
- **테마 선택**: 4가지 디자인 스타일 중 선택
  - 미니멀/모던
  - 레트로 게임
  - 네온/사이버펑크
  - 귀여운/캐주얼
- **시작하기 버튼**: 클릭 시 두 번째 페이지로 이동

---

### 2️⃣ 두 번째 페이지 - 테스트 화면

#### 게임 플로우
1. **대기 상태**: 커다란 빨간색 원 버튼 표시
2. **깜빡임 효과**: 빨간색 원이 fade in/out으로 사라졌다 나타남 반복
3. **전환**: 2초 ~ 5초 사이 랜덤 시간 후 초록색 원으로 변경
4. **측정**: 초록색 원이 나타난 순간부터 사용자가 클릭할 때까지의 시간 측정
5. **결과 표시**: 반응 시간(ms)을 초록 버튼 위에 표시
6. **카운트다운**: 3, 2, 1 시각적 카운트다운 표시
7. **반복**: 카운트다운 후 다시 1번부터 시작 (총 2회 테스트)
8. **완료**: 2회 테스트 완료 후 세 번째 페이지로 자동 이동

#### 미스클릭 처리
- 빨간색 원 상태에서 클릭 시:
  - "너무 빨라요! 다시 탭하세요." 메시지 표시
  - 해당 라운드 처음부터 다시 시작

---

### 3️⃣ 세 번째 페이지 - 결과 화면

#### 구성 요소
- **개별 결과**: 1차, 2차 반응 시간 각각 표시
- **평균 속도**: 두 번의 테스트 평균값 표시
- **등급 시스템**: 별점으로 시각적 표시

#### 등급 기준 (평균 속도 기준)
| 등급 | 조건 | 표시 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 200ms 이하 | 별 5개 |
| ⭐⭐⭐⭐ | 200ms 초과 ~ 260ms 이하 | 별 4개 |
| ⭐⭐⭐ | 260ms 초과 ~ 350ms 이하 | 별 3개 |
| ⭐⭐ | 350ms 초과 ~ 500ms 이하 | 별 2개 |
| ⭐ | 500ms 초과 | 별 1개 |

#### 버튼
- **다시 도전하기**: 두 번째 페이지로 이동 (동일 테마 유지)
- **처음으로**: 첫 번째 페이지로 이동 (테마 재선택 가능)

---

## 디자인 테마

### 테마 1: 미니멀/모던
- 깔끔한 흰색/회색 배경
- 심플한 원형 버튼
- 산세리프 폰트
- 부드러운 그림자 효과

### 테마 2: 레트로 게임
- 픽셀 아트 스타일
- 8bit 컬러 팔레트
- 픽셀 폰트
- 도트 패턴 배경

### 테마 3: 네온/사이버펑크
- 다크 배경
- 네온 글로우 효과
- 그라데이션 색상
- 미래지향적 폰트

### 테마 4: 귀여운/캐주얼
- 파스텔 톤 색상
- 둥근 모서리
- 귀여운 아이콘/이모지
- 친근한 폰트

---

## 기술 상세

### 상태 관리
- React useState/useContext로 관리
- 테마 선택, 반응 시간 결과 등 저장

### 타이밍 측정
- `performance.now()` 사용으로 정밀한 시간 측정
- 밀리초(ms) 단위로 표시

### 애니메이션
- CSS 애니메이션 또는 Framer Motion 사용
- Fade in/out 효과
- 카운트다운 애니메이션

### 반응형 디자인
- 모바일 우선 설계
- 터치 이벤트 지원
- 다양한 화면 크기 대응

---

## 배포 고려사항

- Static Export 설정 필요
- `next.config.js`에 `output: 'export'` 설정
- 서버 사이드 기능 미사용
