영웅은 역사적인 인물을 모에화한 2D 인물을 뜻합니다.
이순신, 이성계, 척준경, 김유신 등이 있습니다.

각 영웅 마다 능력치가 있다.
 영웅 두 명을 매치 시키면 각 영웅의 능력치를 FM(풋볼매니저) 처럼 보여 준다. 
영웅이 매치 되서 결투를 할 경우 누가 이기는지 표현한다. 
LLM 으로 각각의 초상화, 20 명의 영웅의 경우 400개의 매치 일러스트를 그려야 한다. ( 20x20)
 LLL으로 미리 각 영웅의 능력치과 각 영웅의 개요를 입력 한다.
 LLM 으로 영웅의 전투 결과를 미리 시뮬레이션을 한다. ( 20명의 경우  400개 )
 이때 랜덤하게 경우의 수를 만들어도 좋을 것 같다.( 이기는 경우, 지는 경우 이 경우 800개로 됨 )
 똑같은 영우의 배틀 매치에도 다른 결과가 나오는 거다.
일단 초기버전은 20 명만 해서 스트리트파이터에서 1P와 2P를 선택하는 것처럼 꾸며도 좋을 듯 하다

초상화로 사용할 작은 거 하나.
초상화를 클릭할 때 나올 전체 일러스터 하나
붙었을때 매치 하는 결과 하나 둘이 전투 하는 거 하나, 한쪽이 이기는 거 각각 하나
 노벨 AI를 통해서 레퍼런스 이미지를 출력 한다.
 그리고 그 레퍼런스 이미지를 참조하여 제미나이로 실제 초상화와 전투 장면 전체 일러스트,
 이 3개를 뽑는다.
 노벨 AI를 써서 전체 일러스트 와 초상화를 뽑아도 된다. 
하지만 전투 화면은 제미나이를 통해 일관성을 유지한다.

배틀 할 때 한쪽이 이길 확률을 전투력을 기반으로 하면 좋을듯. 전투력 90과 전투력 10이 붙으면 90% 확률로 왼쪽이 이겨야지.
지금 고민 되는 게 이민지는 r2 버킷에 올린다 하더라도, 각 영웅들의 개요나 전투 했을 때에 대한 텍스트는 어떻게 해야 될지 고민이다.
어떤 기술 스텍에 써야될지 정해야한다.
