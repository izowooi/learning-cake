# 사다리타기 웹 앱 PRD (Product Requirements Document)

## 1. 개요
사다리타기 웹 앱을 만드려고 합니다. 참가자들이 이름과 경품을 입력하고, 랜덤으로 생성된 사다리를 통해 결과를 확인할 수 있는 인터랙티브 게임입니다.

---

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| 프레임워크 | Next.js (정적 빌드, `output: 'export'`) |
| 스타일링 | Tailwind CSS |
| 배포 | Cloudflare Pages |
| 데이터베이스 | 없음 (클라이언트 전용) |

---

## 3. 페이지 구조

### 3.1 헤더 영역
- "사다리타기"라는 큰 제목 표시

### 3.2 설정 영역
- **참가자 수 설정**
  - 최소: 2명
  - 최대: 15명 (권장)
  - 기본값: 5명
  - 숫자 입력 또는 +/- 버튼으로 조절
  - "적용" 버튼을 누르면 사다리 그림에 반영

- **초기화 버튼**
  - 참가자 수 설정 바로 다음 줄에 위치
  - 입력했던 이름과 경품 내용만 초기화
  - 참가자 수(직선 개수)는 유지됨

### 3.3 사다리 영역

#### 상단 입력 칸 (참가자 이름)
- 각 수직선 위쪽에 입력 칸 배치
- 참가자 이름 입력 가능
- 미입력 시 빈칸으로 유지

#### 사다리 그림
- 참가자 수만큼 수직으로 평행한 직선 표시
- 예: 2명 → 2개의 수직선, 3명 → 3개의 수직선
- 그림 영역의 크기는 참가자 수와 관계없이 동일하게 유지

#### 하단 입력 칸 (경품/결과)
- 각 수직선 아래쪽에 입력 칸 배치
- 상품이나 경품 내용 입력 가능
- 미입력 시 빈칸으로 유지

### 3.4 시작 버튼
- 사다리 그림 아래에 위치
- 버튼 클릭 시 게임 실행

---

## 4. 사다리 로직

### 4.1 가로선(다리) 생성
- **생성 시점**: 시작 버튼 클릭 시
- **생성 방식**: 랜덤으로 자동 생성
- 인접한 수직선 사이에 가로선 배치
- 같은 높이에 연속된 가로선은 생성되지 않음 (사다리 규칙 준수)

### 4.2 경로 계산
- 각 참가자의 시작점에서 아래로 이동
- 가로선을 만나면 해당 방향으로 이동
- 최종 도착 지점의 경품과 매칭

---

## 5. 애니메이션 및 결과 표시

### 5.1 애니메이션
- 시작 버튼 클릭 시 모든 참가자가 **동시에** 사다리를 타는 애니메이션 실행
- **시각적 효과**: 지나간 경로의 선이 두꺼워지는 효과
- 각 참가자별로 다른 색상으로 경로 표시 (구분 용이)

### 5.2 결과 표시
- 애니메이션 완료 후 각 참가자와 경품 매칭 결과 표시
- 결과를 명확하게 시각적으로 연결하여 보여줌

---

## 6. 반응형 디자인

### 6.1 지원 디바이스
- 모바일 (320px ~)
- 태블릿 (768px ~)
- 데스크톱 (1024px ~)

### 6.2 반응형 고려사항
- Tailwind CSS 브레이크포인트 활용
- 모바일에서는 사다리가 화면 너비에 맞게 축소
- 입력 칸은 터치 친화적인 크기 유지
- 버튼은 모바일에서 충분한 터치 영역 확보

---

## 7. 사용자 흐름

1. 페이지 접속 → 기본 5명 설정된 사다리 표시
2. (선택) 참가자 수 변경 → 적용 버튼 클릭
3. 상단 입력 칸에 참가자 이름 입력
4. 하단 입력 칸에 경품/결과 입력
5. 시작 버튼 클릭
6. 가로선 랜덤 생성 + 애니메이션 실행
7. 결과 확인
8. (선택) 초기화 버튼으로 재시작

---

## 8. 추후 고려사항 (선택)

- 애니메이션 속도 조절 기능
- 결과 공유 기능 (이미지 저장 또는 링크)
- 다크 모드 지원
- 사운드 효과
