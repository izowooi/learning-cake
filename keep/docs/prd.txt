# Keep - 메모 공유 웹 앱 PRD

## 개요
친구들끼리 사용할 간단한 메모 공유 웹 앱을 만드려고합니다.
전반적인 UI 나 기능은 Google Keep 과 동일했으면 좋겠습니다.
URL 을 아는 누구나 글을 쓸 수 있고, 수정할 수 있어야합니다.
보안에 문제가 생겨도 상관없습니다. 어차피 한시적으로 한달만 사용하고 삭제할거에요.

## 기술 스택
- Frontend: Next.js
- Deployment: Cloudflare Pages
- Backend: Firebase Realtime Database (프로젝트: clever-lemon)

## 데이터 스키마
```
memos/{id}
├── title: string (최대 100자)
├── content: string (최대 5,000자)
├── pinned: boolean
├── createdAt: timestamp
└── updatedAt: timestamp
```

## 정렬 기준
- 즐겨찾기(Pinned) 메모 상단 고정
- 나머지는 수정일(updatedAt) 기준 최신순

## 제약 사항 (DDoS 방어)
혹시라도 디도스 공격을 하면 안 되니, 그 부분만 막아주세요. 벡엔드에 글을 쓰는 부분만 방어처리 하면 될거 같습니다.
예를 들어 너무 큰 크기의 파일을 쓰려고 하거나 너무 비정상적인 행동이면 프론트엔드에서 미리 제어해야해요.
- 제목: 최대 100자
- 본문: 최대 5,000자
- 프론트엔드에서 입력 시 글자 수 제한

## 핵심 기능 (CRUD)

### 1. 읽기
먼저 작성된 메모를 모두 한눈에 스크롤 하면서 볼 수 있어야합니다.
이는 Google Keep 과 동일하며 내용이 너무 긴 경우 그냥 ... 으로 요약해서 보여주세요.
작성된 메모를 누르면 수정단계로 넘어갑니다.

### 2. 수정
수정할 때도 Google Keep 과 동일합니다.
메모를 누르면 수정할 수 있는 UI 가 나오고, (읽기 UI 와 흡사) 수정이 가능합니다.
수정은 따로 제출이 없습니다. 그냥 글을 수정하면 저장되는 거에요.
수정하는 뷰는 크게 타이틀과 본문으로 구성되면 하단에 버튼 2개가 있습니다.
하단에는 닫기, 삭제 두개의 버튼이 존재하며, 상단 타이틀에는 즐겨찾기를 온/오프 하는 기능이 들어갑니다. 구글 킵은 우측에 Pinned 아이콘이 붙어있어요.

### 3. 쓰기
이것도 Google Keep 처럼 최상단에 "메모 작성" 버튼을 만들어주세요.

### 4. 삭제
삭제는 수정하는 뷰에서만 삭제가 가능하도록 해주세요.

## UI/UX 가이드라인
모든 UI 의 룩앤필은 미니멀리즘입니다. 실제로 Google Keep 의 UI 를 보면 정말 단촐한 것을 알 수 있어요.
여기에 모던함도 넣으면 좋을 것 같아요. 모던하고 미니멀한 디자인을 추구합니다. 그리고 정말 중요한 점은 모바일에서 주로 사용되기에 모바일 UI 를 고려해주세요.
Google Keep 을 보면 width 에 따라서 배열되는 그리드가 변경됩니다. 저도 이렇게 하고 싶어요.
